## Development Commands

```bash
mise run setup        # Install deps (npm + go mod download)
mise run test         # go test ./...
mise run fmt          # gofmt -w cmd internal
mise run css          # Compile Tailwind CSS
mise run serve        # Start server (runs css first)
mise dev              # Hot-reload dev server on 127.0.0.1:8080
```

To run a single test package:
```bash
go test ./internal/game/...
go test ./internal/session/...
```

## Architecture

Hearts is a multiplayer browser card game. All state is in-memory; nothing persists across restarts.

### Layer boundaries (strict, no skipping)

| Layer | Package | Responsibility |
|---|---|---|
| Domain | `internal/game` | Pure Hearts rules, `game.Round` state machine, card/scoring logic. No I/O. |
| Bot strategies | `internal/game/bot` | `bot.Bot` with `ChoosePlay`/`ChoosePass` methods. Bots hold no player state. |
| Protocol | `internal/protocol` | WebSocket message types (commands + events). Shared contract between table and web clients. |
| Session runtime | `internal/session` | `session.Table` — one goroutine per table (actor pattern). Drives `game.Round`, emits events. |
| HTTP/WS | `internal/webui` | Serves embedded assets, manages WebSocket lifecycle, routes to session runtime. |

### Concurrency model

Each table runs as a single goroutine; all commands are serialized through its channel. WebSocket clients and bots both send messages — there is no special internal access path for bots.

### Key design rules (from `architecture.md`)

- The session runtime is the **only authority**: it drives the `game.Round` state machine and emits events based on results.
- `game.Round` is a step-at-a-time state machine for one round. It owns hands, tricks, passes, and scoring. The session calls `SubmitPass`/`ApplyPasses`/`MarkReady`/`StartPlaying`/`Play` and reads state via query methods.
- `game.TurnInput`/`PassInput` are pure domain types for decision context.
- Bot detection: `player.bot != nil` in `playerState`.
- Human reconnection works via a browser token/cookie (within the running process lifetime only).
- Logging uses `slog` with structured JSON.

### Assets

Frontend assets (HTML, CSS, JS, SVG cards) are embedded into the binary via `//go:embed`. CSS is generated by Tailwind from `webui/assets/styles.input.css` — edit the input file, not the generated `styles.css`.

### Asset fingerprinting

HTML pages are Go `html/template` files with `{{.StylesURL}}`, `{{.ScriptURL}}`, and `{{.ExtraScripts}}` placeholders for asset URLs. Templates are rendered once at startup with either fingerprinted or plain URLs depending on mode. In production, CSS and JS are served at content-hashed URLs (e.g. `/assets/styles.a1b2c3d4.css`) with `Cache-Control: public, max-age=31536000, immutable`; plain CSS/JS paths return 404. JS module imports are rewritten to use fingerprinted filenames. In dev mode (`-dev` flag / `mise dev`), assets are served at plain paths with no cache headers so hot-reload works normally. See `internal/webui/fingerprint.go`.
